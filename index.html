Desenvolvimento de sistemas

O curso de desenvolvimento tem por objetivo formar técnicos capazes de criar páginas WEB e programas de computadores e outras máquinas, como celulares. O primeiro semestre, centrado nas informações mais básicas, é constituido de quatro matérias cujos conteúdos encontram-se resumidos abaixo.

Fundamentos de Prog. Orientada a Objeto

O que é?

Apertar botões, calcular coisas, filtrar informações... As máquinas nos permitem fazer muitas coisas, mas quem ensinou tantas coisas a elas? Tenha certeza de que elas não “nasceram” sabendo, mas sim, “aprenderam” com a ajuda de muito e muitos humanos.

Esse processo de “dizer” à máquina o que fazer se dá por meio de códigos e quem os escreve são os programadores. Esses profissionais têm que ser fluentes no “código” para que possam escrever uma receita com todos – repito: todos! – os passos que o programa precisa para concluir uma tarefa. Uma vez descrito o processo, a máquina refazê-lo várias e várias vezes.

Como escrever receitas?

Atualmente, existem muitas linguagens de programação, que nada mais são do que a forma de escrever a receita. Depois de escrita, essa receita passa por uma compilação, processo que traduz tudinho para a “língua” das máquinas, mais especificamente para os SOs.

A linguagem Java tem, por sua vez, duas compilações: ela traduz o código fonte feito pelo programador para um bytecode, que é lido por um programa Java presente na máquina. Este programa, chamado JRE ou JVM, compila novamente o bytecode, dando como resultado é algo que o SO envolvido consegue entender. Esse processo é bom, pois um mesmo código fonte e seu respectivo bytecode podem ser “lidos” por diversos SOs. Ao lado, uma esquematização desse processo.

Aprendemos nesta matéria, portanto, passar a receita para a máquina usando Java. A nossa lógica foi construída com base em uma programação orientada a objetos, que objetiva “trazer” para a programação a nossa realidade, tornando o código mais concreto.

Sistemas Operacionais

O que é?

Estamos constantemente em contato com a tecnologia, manipulando-a à nossa vontade para diversos fins. Você já parou para pensar o que possibilita que essa manipulação ocorra?

Como resposta a essa pergunta, temos os Sistemas Operacionais. Eles são uma parte não-física, conhecida como Software, que nos ajuda a comandar a parte física, chamada Hardware, atuando, portanto, como intermediadores, tradutores de informações.

Eles nem sempre existiram: no início, o contato humano-hardware era direto, então qualquer deslise era fatal. Assim, o SO surgiu para nos ajudar preservar as informações e ferramentas tecnológicas.

Eles estão presentes em computadores, smartwatch, videogames, carros e até em máquinas de lavar e são divididos em Kernel, que é o coração, que entende a “língua” do hardware, e o restante que está mais “próximo” ao usuário. Os exemplos mais conhecidos de SO são o Windows, Linux, MacOs e Android.

Hardware e Redes

O que é?

Muito já foi falado sobre o hardware das máquinas. Esta matéria nos ajuda a aprender um pouco mais sobre essa parte. Em HARE, falamos dobre Hardware e também sobre Redes.

O hardware da máquina é tudo aquilo que podemos tocar e, para tudo funcionar é necessário que haja um componente que conecte e controle todo mundo: a placa-mãe é quem exerce essa função. Ela recebe energia elétrica de uma fonte ligada a uma tomada e distribui essa energia para todos os seus “filhos”, além de intermediar a “conversa” entre eles.

Se a placa-mãe ajuda a conectar os componentes de uma máquina, o que faz uma máquina se conectar a outra? Essa pergunta é respondida na matéria de Redes. Uma máquina pode se conectar a uma outra através de cabos ou não ou ainda pode se conectar a diversas outras máquinas, formando uma rede que respeita uma tipologia. Nessas conexões, alguns endereços são usados para identificar uma máquina, como o IP e o Mac.

Linguagem de Marcação

O que é?

Quantas vezes você usa o Google por dia? E em uma semana? Já parou para o pensar como tudo aquilo foi parar ali e daquela forma tão organizada?

Essa organização não acontece sozinha: mais uma vez nós humanos precisamos “contar” para a máquina o que é cada coisa e onde cada uma delas deve ficar. Para isso, marcamos os textos, as imagens e outros recursos e usamos essa marca para estabelecer os critérios e assim criar as páginas da internet.

Se há uma linguagem para programar, também há uma para organizar: as linguagens de marcação . A abordada no curso é a HTML. Através de suas tags, é possível indicar os significados de cada uma das informações presentes na tela e, com ajuda do CSS é possível deixar o mundo colorido! (e não só isso: mais organizado, proporcional e esteticamente agradável).

Oie?! "Marcar" como?

Como já exposto, é necessário marcar as informações de acordo com sua função. Essa marcação se dá com o uso das tags, que podem ser entendidas como etiquetas identificadoras. Assim, podemos dizer que algo é:

  • - um título principal (h1);
  • - um título secundário, terciário... (h2, h3..);
  • - um paragrafo (p);
  • - uma imagem (img);
  • - uma lista ou um item de uma lista (ul, li);

Além disso, podemos definir as “caixinhas” que “guardarão” cada um dos itens com o uso das divs , por exemplo. Para você ter uma ideia, tudo aqui nesta página foi feito usando essa linguagem! Legal né?!
